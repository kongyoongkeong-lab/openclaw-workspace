# Code of Conduct

## Scope

This repository supports Jason's OpenClaw workspace, memory structure,
automation setup, and GitHub governance baseline.

Participation in issues, pull requests, reviews, discussions, and related
automation should stay technical, respectful, and focused on improving the
workspace safely.

## Expected Behavior

- Keep feedback specific, actionable, and evidence-based.
- Respect privacy boundaries around tokens, credentials, personal memory,
  browser sessions, and runtime logs.
- Avoid publishing private operational details unless Jason has explicitly
  approved them for GitHub.
- Use small, reviewable changes for workspace setup, automation, memory, and
  governance files.
- Call out security, privacy, or production-risk concerns directly.

## Unacceptable Behavior

- Sharing secrets, credentials, OAuth files, private memory logs, or raw runtime
  dumps.
- Harassment, personal attacks, or disruptive conduct in review threads.
- Bypassing stated review gates, branch protections, or safety invariants.
- Submitting broad restore, merge, or generated-artifact changes that obscure
  what is being changed.

## Enforcement

Jason and the Pentagon `@main` orchestrator may close, edit, or reject issues,
pull requests, comments, or automation output that violates this policy.

Security-sensitive content should be removed from public discussion and handled
through the repository security process in `SECURITY.md`.
