# Contributing to OpenClaw Workspace

## Project Structure

- **pentagon/** - Pentagon agent team hierarchy
- **memory/** - Runtime memory logs
- **tools/** - Local tooling configurations
- **assets/** - GPU-accelerated assets and media

## Development Workflow

1. Feature branches from `develop`
2. PRs into `main` for releases
3. CI/CD handles automated testing

## Code Style

- Python: Black formatter
- JavaScript: Prettier
- Git: Conventional Commits

## Security

- No secrets in git
- Use `.env` for API keys
- Follow @sentinel guardrails

## Getting Help

- Check HEARTBEAT.md for system status
- Review AGENTS.md for team structure
- Contact team lead for escalations
