## ERR-20260524-001 — Poster AI Copy Timeout

**Date:** 2026-05-24
**Priority:** medium
**Status:** open
**Area:** image
**Detected By:** @main

### Summary
Poster generation aborted when local Ollama AI-copy generation timed out.

### Trigger
- Command/workflow: `tools/make_poster.py --ai-copy --auto-color`
- Input class: Telegram `做海报` request for a travel/person beach portrait
- Environment: local Ollama `qwen3.5:9b`

### Error Signal
`ollama run qwen3.5:9b` timed out after 60 seconds inside `tools/local_ai_copy.py`.

### Root Cause
AI-copy was a hard dependency in the poster path, so a local LLM timeout stopped the entire render pipeline.

### Recovery
Re-ran P5.9 without `--ai-copy`, kept `--auto-color`, and generated the poster successfully.

### Prevention Rule
Poster workflows must degrade to deterministic copy/rendering when AI-copy times out. Consider pre-warming Ollama or increasing timeout only when VRAM and latency budget are safe.

### Related
- Files: `tools/make_poster.py`, `tools/local_ai_copy.py`, `tools/local_poster_premium.py`
- Services: Ollama

## ERR-20260524-003 — WSL PowerShell Quoting And Reserved Variables

**Date:** 2026-05-24
**Priority:** medium
**Status:** mitigated
**Area:** windows
**Detected By:** @main

### Summary
WSL-to-Windows UI automation failed from Bash expansion and PowerShell reserved variable use.

### Trigger
- Command/workflow: Doubao Windows UI automation bridge
- Input class: PowerShell command from WSL
- Environment: WSL2 calling Windows PowerShell

### Error Signal
Bash expanded PowerShell `$_` inside a double-quoted command, producing invalid process references. PowerShell script also used `$pid`, conflicting with read-only `$PID`.

### Root Cause
Mixed-shell quoting was not isolated, and the script used a reserved PowerShell variable name.

### Recovery
Wrapped PowerShell commands in single quotes or escaped `$`; renamed script variable to `$processId`.

### Prevention Rule
Keep WSL-to-PowerShell examples in `tools/windows_ui/README.md` using `-Command "& '<UNC script>' ..."` and avoid reserved PowerShell variable names.

### Related
- Files: `tools/windows_ui/README.md`
- Services: Windows UI Automation bridge

## ERR-20260525-003 — n8n Webhook Returned Start Acknowledgement Only

**Date:** 2026-05-25
**Priority:** medium
**Status:** fixed
**Area:** n8n
**Detected By:** @main

### Summary
The n8n PDF-to-PPTX webhook returned `Workflow was started` instead of the workflow's JSON payload.

### Trigger
- Command/workflow: `POST http://localhost:5678/webhook/pdf-to-pptx`
- Input class: JSON body with `file_path`
- Environment: n8n `2.21.7`

### Error Signal
Webhook response was only a start acknowledgement, and previous executions had errors.

### Root Cause
The Webhook node was not configured with `responseMode=responseNode`, and the Code node used a Python template incompatible with the active execution path.

### Recovery
Patched workflow JSON to set `responseMode=responseNode`, converted the code node to JavaScript, restarted n8n, and verified JSON response.

### Prevention Rule
For n8n workflows expected to return a response body, explicitly set Webhook `responseMode=responseNode` and smoke-test the production webhook after activation/restart.

### Related
- Files: `n8n/create_simple_workflow.sh`, `n8n/docker-compose.yml`
- Services: n8n
